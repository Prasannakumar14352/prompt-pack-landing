EMAIL DELIVERY SETUP GUIDE
claudepromptpack.in
===================================================

OVERVIEW
--------
After a buyer pays, two things happen automatically:
1. They land on /thank-you with immediate download buttons
2. An email is sent to them with the same download links

This guide covers setting up the email delivery using n8n
(which you already know) OR Make.com.

===================================================
STEP 2A: EMAIL DELIVERY VIA RAZORPAY WEBHOOK + n8n
===================================================

--- RAZORPAY WEBHOOK SETUP ---
1. Razorpay Dashboard → Settings → Webhooks → Add New Webhook
2. Webhook URL: https://your-n8n-instance.com/webhook/razorpay-payment
3. Secret: create a random string, save it
4. Events to subscribe: payment.captured
5. Save

--- n8n WORKFLOW ---
Node 1: Webhook (POST)
  - Path: /razorpay-payment
  - Authentication: None (we validate signature manually)

Node 2: Function (validate Razorpay signature)
  const crypto = require('crypto');
  const secret = 'YOUR_WEBHOOK_SECRET';
  const body = $input.first().json.rawBody;
  const sig = $input.first().headers['x-razorpay-signature'];
  const expected = crypto.createHmac('sha256', secret).update(body).digest('hex');
  if (expected !== sig) throw new Error('Invalid signature');
  return $input.all();

Node 3: Set (extract payment details)
  - email:  {{ $json.payload.payment.entity.email }}
  - amount: {{ $json.payload.payment.entity.amount }}
  - name:   {{ $json.payload.payment.entity.contact }}
  - plan:   {{ $json.payload.payment.entity.description.includes('Bundle') ? 'bundle' : 'base' }}

Node 4: IF (check plan)
  - Condition: {{ $json.plan }} equals "bundle"
  - True path  → Send bundle email
  - False path → Send base email

Node 5a: Send Email — Gmail/SMTP (Base plan)
  To: {{ $json.email }}
  Subject: 🎉 Your Claude Prompt Pack is ready to download!
  Body: [paste BASE_EMAIL_TEMPLATE from below]

Node 5b: Send Email — Gmail/SMTP (Bundle plan)
  To: {{ $json.email }}
  Subject: 🚀 Your Complete AI Bundle is ready — 8,360+ prompts inside!
  Body: [paste BUNDLE_EMAIL_TEMPLATE from below]

===================================================
STEP 2B: EMAIL DELIVERY VIA PAYPAL WEBHOOK + n8n
===================================================

--- PAYPAL WEBHOOK SETUP ---
1. PayPal Developer Dashboard → My Apps → Your App → Webhooks
2. Add webhook URL: https://your-n8n.com/webhook/paypal-payment
3. Events: PAYMENT.CAPTURE.COMPLETED
4. Save

--- n8n WORKFLOW ---
Node 1: Webhook (POST) — path: /paypal-payment

Node 2: HTTP Request (verify with PayPal)
  URL: https://api.paypal.com/v1/notifications/verify-webhook-signature
  Method: POST
  Auth: Bearer YOUR_PAYPAL_ACCESS_TOKEN
  Body: {
    "transmission_id": "{{ $json.headers['paypal-transmission-id'] }}",
    "transmission_time": "{{ $json.headers['paypal-transmission-time'] }}",
    "cert_url": "{{ $json.headers['paypal-cert-url'] }}",
    "auth_algo": "{{ $json.headers['paypal-auth-algo'] }}",
    "transmission_sig": "{{ $json.headers['paypal-transmission-sig'] }}",
    "webhook_id": "YOUR_WEBHOOK_ID",
    "webhook_event": {{ $json.body }}
  }

Node 3: IF — verification_status = SUCCESS

Node 4: Set (extract details)
  - email:  {{ $json.body.resource.payer.email_address }}
  - name:   {{ $json.body.resource.payer.name.given_name }}
  - amount: {{ $json.body.resource.amount.value }}
  - plan:   {{ parseFloat($json.body.resource.amount.value) >= 19 ? 'bundle' : 'base' }}

Node 5: IF plan = bundle → Send bundle email, else send base email

(same email templates as above)

===================================================
SECURITY NOTES
===================================================

- Google Drive "Anyone with link" links are unguessable but not
  truly private. For a ₹399 product this is acceptable.
  
- If you want stronger protection, use a signed URL service
  (AWS S3 pre-signed URLs, Cloudflare R2) — ask me to set this up
  if needed.

- Do NOT put download links in the public landing page HTML.
  They should only appear on the thank-you page (after payment).

- The thank-you page URL itself (/thank-you?plan=base) is guessable,
  but without your actual Google Drive links in it, it's just an empty page.
  The links are populated from FILES object in the JS — which is in the page.
  So the real protection is the unguessable Google Drive file IDs.
